O GitHub é como uma “rede social” para programadores, mas em vez de postar fotos ou mensagens, você posta códigos.
- Ele guarda seus projetos de programação na nuvem, assim você pode acessar de qualquer lugar.
- Permite que várias pessoas trabalhem juntas no mesmo projeto sem bagunça, cada uma fazendo suas partes e depois juntando tudo.
- Funciona como uma vitrine: você pode mostrar seus trabalhos para outras pessoas, empresas ou colegas.
- Além disso, você pode olhar projetos de outros, aprender com eles e até colaborar.
Pense no GitHub como um Google Drive especial para códigos, mas com ferramentas que ajudam a organizar, controlar versões e trabalhar em equipe.

Fluxo local: do zero ao primeiro commit
- Iniciar o repositório
- Comando: git init
- O que acontece: Cria a pasta .git com todo o mecanismo de histórico.
- Ver status das alterações
- Comando: git status
- O que acontece: Mostra arquivos modificados, não rastreados e o branch atual.
- Adicionar mudanças ao stage
- Comando: git add arquivo.ext ou git add .
- O que acontece: Seleciona exatamente o que entrará no próximo commit (curadoria das mudanças).
- Criar um commit com mensagem clara
- Comando: git commit -m "mensagem objetiva"
- O que acontece: Registra um “snapshot” do projeto com contexto.
- Revisar histórico
- Comando: git log --oneline --graph --decorate
- O que acontece: Visualiza commits e branches de forma compacta.
